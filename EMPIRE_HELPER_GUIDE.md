## 2. 앱 UI 레이아웃 및 기능 구성 (App Structure)

### 📍 프론트엔드 구성 요소
- **플로팅 액션 버튼 (FAB)**: 화면 우측 하단에 위치하며 클릭 시 챗봇 대화창이 슬라이드 업 됩니다.
- **엠파이어 도우미 툴팁**: 챗봇 아이콘 위에 항상 노출되거나 호버 시 나타나며, "관리규약 기반의 정확한 정보를 안내해 드립니다"라는 문구를 포함합니다.

### 퀵 액션 카드 (Quick Menu)
1. **🚗 주차 관리**: 요금 조회, 차량 변경 신청, 할인권 잔여 확인.
2. **📅 시설 예약**: 공용회의실 실시간 예약 및 결제 현황.
3. **📑 관리규약**: 2025년 최신 규약 전문 열람 및 검색.
4. **🔔 공지사항**: 협의회 의결 사항 및 회계감사 결과 보고.

## 3. 관리규약 vs 집합건물법 비교 로직 (Database)

챗봇이 답변 시 활용할 내부 데이터 테이블 구조입니다.

| 구분 | 관리규약 내용 (시스템 로직) | 대한민국 집합건물법 (법적 근거) |
| :--- | :--- | :--- |
| **의결권 행사** | 소유 지분에 따라 행사하며, 공유 시 1인 선임 필수 | **제37조**: 각 구분소유자의 의결권은 지분 비율에 따름 |
| **규약 개정** | 의결권의 3/4 이상 찬성으로 설정·변경·폐지 | **제29조**: 규약 설정·변경은 관리단집회 3/4 이상 찬성 필요 |
| **관리인 선임** | 협의회장 또는 관리업체 책임자가 관리 업무 수행 | **제24조**: 구분소유자 10인 이상 시 관리인 선임 의무 |
| **연체자 제재** | 3개월 연체 시 서비스 제한 및 법적 조치 (단전·단수 등) | **제25조**: 관리인은 관리비 징수 및 보존 행위 권한을 가짐 |
